# Система учёта экскурсий "Соната-Крым"

## Описание
Веб-приложение для автоматизации работы экскурсионного агентства с функциями:
- Продажа экскурсий с учётом групп туристов
- Управление долгом (частичная оплата)
- Формирование списков для водителей
- Учёт наличных денег у реализаторов
- Общий чат для команды
- Импорт/экспорт данных в Excel

## Установка на сервер Beget

### 1. Подготовка
```bash
# Перейдите в директорию проекта
cd ~/sonata.tagir75.ru

# Установите зависимости
npm install
```

### 2. Инициализация базы данных
База данных создаётся автоматически при первом запуске.

### 3. Создание первого пользователя (менеджера)
Запустите сервер и выполните SQL-запрос через SQLite:
```sql
INSERT INTO users (username, password, role, full_name) 
VALUES ('admin', 'admin123', 'manager', 'Администратор');
```

### 4. Запуск сервера
```bash
# В режиме разработки
node server.js

# В продакшене (через PM2)
npm install -g pm2
pm2 start server.js --name sonata-crm
pm2 save
pm2 startup
```

### 5. Настройка HTTPS
На Beget SSL-сертификат настраивается в панели управления хостингом.

### 6. Настройка домена
Убедитесь, что домен sonata.tagir75.ru указывает на папку с проектом.

## Роли пользователей

### Менеджер (admin)
- Полный доступ ко всем функциям
- Управление справочниками (экскурсии, водители, экскурсоводы и т.д.)
- Создание активных экскурсий на даты
- Просмотр кассы всех реализаторов
- Обнуление кассы после сбора наличности
- Импорт/экспорт данных

### Реализатор (seller)
- Продажа экскурсий
- Просмотр количества проданных билетов
- Работа с кассой (только своей)
- Участие в общем чате

### Агент (agent)
- Вход по названию гостиницы/точки
- Аналогичные права реализатора
- Отдельный учёт кассы

## Основные функции

### Продажа экскурсии
1. Выбрать экскурсию и дату
2. Ввести фамилию туриста (для группы — одна фамилия)
3. Указать количество человек
4. Выбрать место посадки (время подставляется автоматически)
5. Ввести общую сумму и принятую сумму
6. Долг рассчитывается автоматически
7. Добавить примечание (способ оплаты, рассадка и т.д.)

**Важно:** При продаже группы из N человек создаётся N записей в базе. Долг указывается только на первой записи.

### Печать списка для водителя
1. Выбрать дату
2. Нажать "Печать списка водителю"
3. Список сортируется по порядку точек посадки
4. Долг отображается только у первого туриста в группе

### Касса
- Реализатор видит свою текущую наличность
- Менеджер видит кассу всех реализаторов
- Кнопка "Обнулить" сбрасывает сумму после сбора наличности собственником
- История продаж сохраняется

### Чат
- Общие сообщения для всех пользователей
- Сообщения менеджера выделяются цветом
- Автообновление каждые 10 секунд
- Сообщения хранятся бессрочно

### Импорт/экспорт Excel
Формат файла:
- Лист "Экскурсии": name, description, base_price
- Лист "ТочкиПродаж": name, address
- Лист "Водители": full_name, phone, car_info
- Лист "Экскурсоводы": full_name, phone
- Лист "Пользователи": username, password, role, full_name, point_id
- Лист "Агенты": name, password, point_id

**Важно:** Импорт только добавляет новые записи (дубли пропускаются). Удаление возможно только через интерфейс администратора.

## API endpoints

### Авторизация
- POST /api/auth/login — вход
- POST /api/auth/logout — выход
- GET /api/auth/check — проверка сессии

### Продажи
- GET /api/sales/active — список активных экскурсий
- POST /api/sales — создание продажи
- GET /api/sales/excursion/:id — продажи по экскурсии
- GET /api/sales/stats/:id — статистика по реализации

### Администрирование (только менеджер)
- GET /api/admin/data — все справочники
- POST /api/admin/excursions — добавить экскурсию
- DELETE /api/admin/excursions/:id — деактивировать экскурсию
- ... (аналогично для других справочников)

### Чат
- GET /api/chat — получить сообщения
- POST /api/chat — отправить сообщение

### Касса
- GET /api/cashbox — касса всех (менеджер)
- GET /api/cashbox/me — моя касса
- POST /api/cashbox/reset — обнулить кассу

### Импорт/экспорт
- POST /api/import — импорт Excel
- GET /api/import/export — экспорт Excel

## Обновление данных каждые 10 секунд
Автоматически обновляются:
- Список активных экскурсий (количество проданных билетов)
- Чат
- Касса

Редактирование данных происходит только по кнопке "Сохранить".

## Требования к серверу
- Node.js 14+
- SQLite3
- HTTPS (SSL-сертификат)

## Техническая поддержка
Все данные хранятся в файле `db/sonata.db`. Регулярно делайте резервные копии через экспорт Excel или копирование файла БД.
